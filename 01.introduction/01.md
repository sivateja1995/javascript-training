## Introduction of the javascript language

1. javascript is also known as 'live language' or 'scripting language'.
2. it was created by Brendan Eich in 1995.
3. javascript is a high-level, interpreted programming language.
4. it is primarily used for enhancing the interactivity of web pages.
5. javascript can be run on both client-side (in the browser) and server-side (using environments like Node.js).
6. it supports multiple programming paradigms, including procedural, object-oriented, and functional programming.
7. javascript is dynamically typed, meaning variable types are determined at runtime.
8. it has a rich ecosystem with numerous libraries and frameworks such as React, Angular, and Vue.js.
9. javascript is standardized under the ECMAScript specification.
10. it is one of the core technologies of the World Wide Web, alongside HTML and CSS.

## javascript interpreted or compiled language?

interpreted vs compiled languages:

interpreted languages are executed line-by-line by an interpreter at runtime,while compiled languages are transformed into machine code by a compiler before execution.


examples of interpreted languages: javascript, python, ruby   
examples of compiled languages: c, c++, rust

` 
javascript is primarily an interpreted language, meaning that its code is executed line-by-line by an interpreter at runtime, 
rather than being compiled into machine code beforehand. However, modern JavaScript engines, like Google's V8, use Just-In-Time   
(JIT) compilation techniques to improve performance by compiling frequently executed code into machine code on the fly. 
This hybrid approach allows JavaScript to benefit from both interpretation and compilation.
despite this, javascript is still generally classified as an interpreted language due to its dynamic nature and 
the way it is typically executed in web browsers. `

v8 engine used by google chrome and node.js is an example of jit compilation in javascript.
javascript is dynamically typed and interpreted language.

few more examples of the js engines are v8 google, chakra by IE,spidermonkey by mozilla firefox, javascriptcore by safari.

how javascript engine works?

1. parsing: the engine reads the javascript code and converts it into an Abstract Syntax Tree (AST).
2. compilation: the AST is then compiled into bytecode or machine code using Just-In-Time (JIT) compilation.
3. execution: the compiled code is executed by the engine, performing the desired operations.

summary:  
javascript is a high-level, interpreted programming language primarily used for web development.  
it supports multiple programming paradigms and is dynamically typed.  
modern javascript engines use JIT compilation to enhance performance while maintaining its interpreted nature.
